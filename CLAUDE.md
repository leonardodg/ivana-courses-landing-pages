## Project Overview

This is a frontend website for Ivana Academy's course landing pages, built with React 19, Vite 6, Tailwind CSS v4, and TypeScript. It's designed as a Multi-Page Application (MPA) with a DevContainer environment for consistent development. The site is deployed to GitHub Pages.

## Development Environment

- Uses DevContainer for consistent development environment
- Built with React 19 + Vite 6 + Tailwind CSS v4 + TypeScript
- Nginx serves the static content in production
- Port 3003 is forwarded for Vite development server access
- Port 8088 is forwarded for Nginx production-like preview

## Key Commands

### Development
- `npm install` - Install dependencies
- `npm run dev` - Start development mode with Vite HMR on port 3003
- `npm run build` - Build the project (compiles HTML/CSS/JS to dist/)
- `npm run clean` - Remove the dist directory

### Code Quality
- `npm run lint` - Run TypeScript compiler to check for type errors

## Architecture Notes

- Uses Vite with React 19 for a fast development experience
- Tailwind CSS v4 integrated as a Vite plugin (no PostCSS config needed)
- Multi-Page Application (MPA) architecture with separate HTML entry points
- Each course has its own landing page (currently main page, can add /candles, /soap, etc.)
- Static site generation with Vite build process that outputs to `dist/` directory
- Nginx serves the static files in production
- GitHub Pages deployment through GitHub Actions (automated)
- Environment variables are managed through .devcontainer/env/*.env files in the DevContainer

## Deployment Process

The site is automatically deployed to GitHub Pages on pushes to the main branch through GitHub Actions workflow.

## Common Development Tasks

1. Making changes to React components in the `src/app/` directory
2. Adding new course landing pages by creating new HTML files in `src/` directory
3. Running `npm run dev` to start the development server with hot module replacement
4. Running `npm run build` to compile changes for production
5. Testing locally with the DevContainer environment

## Additional Information

### DevContainer Configuration
The project uses a multi-file Docker Compose setup:
- `compose/dev.yml` - Development-specific services
- `compose/build-dev.yml` - Build services for development

### Vite Configuration Details
- Configured as a Multi-Page Application (MPA)
- Each HTML file in src/ is treated as a separate entry point
- Outputs hashed filenames for cache busting
- Serves from port 3003 in development with HMR
- Builds to dist/ directory with optimized assets

### Adding New Course Pages
To add a new course landing page:
1. Create a new directory in `src/` (e.g., `src/soap/`)
2. Add an `index.html` file in that directory
3. Register the new page in `vite.config.ts` in the `pages` object
4. Run `npm run build` to generate the new page in dist/

### Environment Variables
Environment variables are managed through the DevContainer configuration and are accessible during the build process.
